# AGENTS.md

Compact guidance for agents working in the Akama repository.

## Build & Verify

- **Always build via `make build`** — OAuth credentials injected via `-ldflags` from `.env` at compile time.
- `.env` must contain: `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`, `GITLAB_CLIENT_ID`, `GITLAB_CLIENT_SECRET` (case-sensitive).
- Quick compile check without OAuth: `go build ./...` (sets `Version = "dev"`; `akama update` refuses dev builds).
- **No test suite exists.** Build verification is `go build ./...` only.
- Agent installers are in `internal/agent/runner.go` (`InstallClaudeCmd`, `InstallOpencodeCmd`). Called during `akama init` or via `akama update_agents` (Telegram command). No `install` CLI subcommand exists.

## Architecture

- **Two modes**, detected in `main.go` before Cobra runs:
  - Normal mode: `cmd.Execute()` → `init`, `start`, `stop`, `status`, `logs`, `restart`, `update`
  - Daemon mode: `--daemon` in `os.Args` → `runDaemon()` → loads config, opens SQLite, recovers interrupted jobs, runs `bot.RunCtx(ctx)`

- **Daemon lifecycle**: `akama start` forks itself via `daemon.ForkDaemon`. The daemon writes its own PID file; `akama stop` sends SIGTERM and polls until PID file disappears (prevents race where `make start` launches a second instance during drain).

- **Package boundaries** (`internal/` prefix = private):
  - `cmd/` — Cobra CLI commands
  - `bot/` — Telegram bot, router, message handling
  - `agent/` — Agent runner registry (claude, opencode), prompt building, output parsing
  - `storage/` — SQLite access, schema migrations, job/conversation/connection CRUD
  - `job/` — Job execution logic, retry wrapper, follow-up handling
  - `config/` — Config loading from `~/.akama/config.yaml`, OAuth variable exports
  - `daemon/` — PID file management, fork logic
  - `git/` — Clone, commit, push, askpass helpers
  - `provider/` — GitHub/GitLab API clients
  - `logger/` — Rotating log writer (daily or 10 MB trigger, keeps 7 gzip archives)

## Key Gotchas

- **`migrations/001_akama.sql` is a stale PostgreSQL draft and is NOT used.** The canonical schema is inline in `internal/storage/db.go` (`migrate()` function), auto-applied on `storage.Open()`.
- **OAuth credentials are NOT in `config.yaml`** — they are baked in at build time via `make build` ldflags. CI uses `OAUTH_GITHUB_*` / `OAUTH_GITLAB_*` secrets (GitHub reserves `GITHUB_` prefix).
- **Config paths with `~/` are expanded at runtime** in `internal/config/config.go` (`expandHome()` called by both `Load()` and `Save()`).
- **Agent output formats differ**: claude outputs single JSON `{"result":"..."}`; opencode outputs NDJSON event stream and exits 0 on API errors (scan for `{"type":"api_error"}` events via `extractOpencodeError()`).
- **Branch names and commit messages are generated by a second agent call** (`agent.GenerateSummary()`) after the main fix, not hardcoded. Falls back to `fix/issue-X` if unparseable.
- **Question detection**: `agent.IsQuestion()` checks if the last non-empty line of agent output ends with `?` — this is the only signal to enter `awaiting_input` state.
- **`GIT_ASKPASS` temp file** lands in OS temp dir (`os.CreateTemp("", "git-askpass-*")`), never inside the repo.
- **Prompt file** written to `<workspacePath>/.akama-prompt.txt` (inside cloned workspace), passed via `-p`, cleaned up after run.

## Concurrency Model

- Each job goroutine registered in both `sync.WaitGroup` (graceful shutdown) and `cancels` map keyed by jobID (per-job cancellation, `internal/job/runner.go`).
- On SIGTERM: daemon context cancelled → `wg.Wait()` drains with 30s hard timeout.
- `job.CancelJob(id)` looks up and calls the `CancelFunc` from the `cancels` map.

## Release & CI

- `.github/workflows/release.yml`: semantic-release creates git tag + GitHub release → builds binaries with version injected via `-X config.Version=$VERSION` → uploads via `gh release upload`.
- **Commits must follow [Conventional Commits](https://www.conventionalcommits.org):** `fix:` → patch, `feat:` → minor, `BREAKING CHANGE:` → major.
- Binaries named `akama-<os>-<arch>`; build matrix only runs when semantic-release cuts a new release.

## Telegram Library (v5)

- `github.com/go-telegram-bot-api/telegram-bot-api/v5` — `GetUpdatesChan` returns only `UpdatesChannel` (no error).
- `AllowedUpdates` must explicitly include `"callback_query"` or inline button taps are silently dropped (`internal/bot/bot.go:69`).
- Callback acknowledgement: `api.Request(tgbotapi.NewCallback(query.ID, ""))`.
- `Bot.ctx` set in `RunCtx` and threaded to all job goroutines.

## Config File

Location: `~/.akama/config.yaml` (created by `akama init`). API keys stored in `api_keys` map (keyed by provider name `anthropic`, `openai`). Automatic migration from old flat `anthropic_api_key`/`openai_api_key` fields on first load (`internal/config/config.go`).
